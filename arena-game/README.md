# Модуль арены — пошаговый боевой движок

Браузерный 3D-движок пошаговых дуэлей в духе «Легенда: Наследие драконов» /
Broken Sun. Работает на десктопе и в Telegram WebApp, без сборки —
чистые ES-модули + three.js (лежит в `vendor/`, CDN не нужен).

## Запуск

**Двойной клик по `start.bat`** — поднимет локальную раздачу файлов и откроет
игру в браузере (свёрнутое чёрное окно — это она; закрыть окно = остановить).
Правки кода видны по F5.

Либо вручную: `python serve.py 8000` и открыть http://localhost:8000.
`serve.py` — обычный статический сервер, но отдаёт файлы с
`Cache-Control: no-store` (без «прилипшего» кэша модулей) и грузит большие
FBX параллельно.

⚠️ Открывать `index.html` двойным кликом (`file:///...`) нельзя и не
получится: браузеры блокируют ES-модули на `file://` (CORS, «нулевой»
origin) — будет вечная «Загрузка бойцов…». Это ограничение браузера, а не
игры; игре сервер не нужен — нужен любой раздатчик статики по http. GitHub
Pages — как раз он, там игра работает без чего-либо дополнительного.

## Структура

```
index.html            оболочка MMORPG: плашка игрока, круглые кнопки навигации,
                      экраны «локация»/«бой» (в бою — слоты эликсиров), нижняя панель
css/game.css          стили оболочки и боевого UI, адаптив под телефон
js/main.js            сборка демо: конфиг бойцов/локаций, режимы, оркестрация боя
js/arena/Arena.js     3D-сцена: рендер, свет, тени, камера, фон, ресайз
js/arena/Fighter.js   боец: загрузка модели, анимации, удар/реакция/смерть
js/arena/BattleSystem.js  чистая боевая логика (3 удара × 3 блока, урон, таймер)
js/arena/BattleUI.js  плашки бойцов, урон/таймер, штурвал атаки/блока,
                      журнал боя и участники (вкладки нижней панели)
js/arena/loaders.js   загрузчики GLB/FBX + склейка "разрезанных" анимаций
assets/models/        модели и анимации
assets/backgrounds/   фоны локаций и арены (svg/jpg/png)
vendor/three/         three.js r160 + загрузчики (офлайн, без CDN)
```

## Два экрана главной панели

**Локация (вне боя)** — картинка местности и кнопки действий: переходы между
локациями и «Охота» (запуск боя). Конфиг — `LOCATIONS` в `js/main.js`:
локация = картинка (или css-градиент) + список действий
(`goto: 'ключ'` — переход, `hunt: true` — бой, без поля — заглушка-тост).

**Бой** — 3D-арена с плашками бойцов (уровень, имя, HP/MP), строкой
«Урон: N», таймером хода, «штурвалом» атаки/блока по центру сцены и пятью
слотами под скиллы/заклинания (пока пустые).

**Нижняя панель с вкладками** общая для обоих экранов: в бою —
«Участники боя», «Лог боя», «Чат», «Список игроков в локации»; вне боя —
только «Чат» и «Список игроков». Чат, аукцион и почта — заглушки,
подключаются отдельными модулями.

## Ключевые идеи движка

**Арена — изолированный модуль.** `new Arena(container)` встраивается в любой
DOM-элемент и сам следит за его размером (ResizeObserver + пересчёт
devicePixelRatio). Интерфейс игры вокруг (аукцион, рюкзак, чат) живёт своей
жизнью — арена о нём не знает.

**Камера никогда не теряет бойцов.** На каждом ресайзе дистанция камеры
пересчитывается так, чтобы оба бойца влезали и по ширине, и по высоте —
и в портрете телефона, и на широком мониторе.

**Любая модель — без подгонки.** Рост бойца нормализуется автоматически:
скелет меряется в позе первого кадра idle и масштабируется до `height`
из конфига, ноги ставятся на землю. Поддерживаются `.glb/.gltf` и `.fbx`.

**Анимации из отдельных файлов.** Анимация (например, удар) может лежать
в другом файле и «надевается» на скелет модели — стандартный пайплайн Mixamo:
один файл с мешем + сколько угодно файлов с анимациями того же рига.
«Разрезанные» на десятки клипов экспорты (Assimp/mixamo-mini) склеиваются
автоматически.

**Бой = события.** `BattleSystem` не знает ни про DOM, ни про 3D: он кидает
события `turnStart / timer / resolve / battleEnd`, а оркестратор в `main.js`
проигрывает анимации и обновляет UI. Эту логику можно перенести на сервер
один в один (для PvP против живых игроков так и нужно сделать).

**Управление боем — 3 × 3.** Удар и блок по трём зонам: вверх / центр / вниз.
Нажатие зоны удара сразу отправляет ход (блок — необязательный переключатель,
выбирается до удара). Кнопки собраны в «штурвал» по центру сцены между
бойцами; журнал боя живёт во вкладке «Лог боя» нижней панели, хранит всю
историю с прокруткой и автоследованием за последней записью.

## Как добавить нового бойца

1. Положите модель в `assets/models/` (GLB с текстурами или FBX
   с встроенными текстурами).
2. Добавьте запись в `FIGHTERS` в `js/main.js`:

```js
orc: {
  name: 'Орк-вожак',
  model: 'assets/models/orc.glb',
  height: 2.1,                      // рост в метрах — масштаб подгонится сам
  stats: { hp: 3000, damage: [200, 280], crit: 0.1, dodge: 0.04 },
  animations: {
    idle:   { file: 'assets/models/orc.glb', clip: 'Idle' }, // clip — имя клипа в файле
    attack: { file: 'assets/models/orc_attack.glb',
              hitTime: 0.5,     // доля клипа, когда удар "доходит" (цифры урона)
              reach: 1.2,       // на сколько метров не добегать до цели при выпаде
              inPlace: true },  // погасить встроенное перемещение по полу (root motion)
  },
},
```

`inPlace: true` нужен для анимаций атак, у которых персонаж сам перемещается
по полу (большинство ударов Mixamo): движением по арене управляет движок
(выпад к цели), иначе встроенный root motion складывается с выпадом и боец
«уезжает». Высота (присед/прыжок) сохраняется.

Если `clip` не указан: единственный клип файла берётся как есть, а множество
безымянных клипов склеивается в один. Анимации из чужого файла работают,
когда имена костей совпадают с костями модели (у всех скачиваний Mixamo
с одного персонажа — совпадают).

Дополнительные анимации (`hit`, `block`, `death`...) добавляются в тот же
словарь и проигрываются через `fighter.play('hit', { once: true })`.
Пока их нет, реакция на удар и смерть сделаны процедурно.

## Экипировка (шмотки) и гардероб

Кнопка «Рюкзак» открывает **гардероб**: отдельная 3D-сцена
(`js/arena/DressingRoom.js`), где персонаж стоит лицом к камере. Модель
можно вращать мышью/пальцем, вкладки переключают игрока/противника.
При надевании вещи персонаж проигрывает анимацию `taunt` (боевой клич),
если она описана в конфиге бойца.

Важно: в примерочной используется **тот же класс Fighter и тот же
`equip()`**, что и на арене, поэтому посадка вещи в бою совпадает с
примерочной 1-в-1. Подбор посадки нового предмета делается на глазах:
надели в гардеробе → поправили `attach` в конфиге → F5.

Предметы описываются в `ITEMS` (`js/main.js`). Надетое синхронизируется
с бойцами на арене и переживает перезапуск боя (желаемое состояние
хранится в `equipState`, бойцы приводятся к нему идемпотентным
`syncEquipment` с очередью — гонки кликов и рестартов не страшны).
Программно: `fighter.equip(ITEMS.bronzeArmor)` / `fighter.unequip('torso')`.

```js
bronzeArmor: {
  name: 'Бронзовый доспех',
  icon: '🛡️',
  slot: 'torso',                    // один слот = один предмет
  model: 'assets/models/bronze_armor.fbx',
  attach: {                         // всё опционально — у слота есть дефолты
    cover: 1.5,                     // размер предмета в долях эталона слота
    scale: 1,                       // ручная поправка размера
    offset: [0, 0, 0],              // [вперёд, вверх, влево], метры
    rotation: [0, 90, 0],           // градусы — если модель «смотрит» не туда
  },
},
```

Как это работает (`js/arena/EquipEngine.js`): движок сам выбирает способ
посадки.

- **Статичная броня/одежда** (Meshy, Tripo — без скелета; слоты torso,
  legs, hands, feet, shoulders) **автоскинится**: движок переносит веса
  скиннинга с меша тела персонажа на вершины предмета (k ближайших вершин
  тела, сварка швов, сглаживание по топологии) — броня гнётся вместе с
  телом в любой анимации, как настоящая одежда. Расчёт делается один раз
  и кэшируется по паре «предмет × персонаж»; повторное надевание мгновенно.
- **Оружие, шлемы, амулеты** (mainhand, offhand, head, belt) жёстко
  крепятся к кости и движутся с ней.
- **Предмет со своим скелетом** (риг, совместимый с Mixamo) перепривязывается
  к скелету персонажа по именам костей — высшее качество, нулевая цена.

Размер подгоняется автоматически под кости конкретного персонажа
(торс шея-бёдра, размах рук и т.д. — см. `SLOT_FIT` в EquipEngine.js),
поэтому один предмет сидит и на маленьком, и на большом бойце. Режим можно
форсировать: `attach: { mode: 'bone' | 'autoskin' }`; качество автоскиннинга
крутится через `attach.knn` (соседей на вершину, 6) и `attach.smooth`
(итераций сглаживания, 2).

Типичная подгонка нового предмета: надеть в гардеробе, посмотреть, поправить
`rotation` (модели часто «смотрят» вбок), затем `cover`/`offset`.

Опционально предмету можно задать свой PBR-материал из отдельных карт:

```js
material: {
  map: '...basecolor.png', normalMap: '...normal.png',
  metalnessMap: '...metallic.png', roughnessMap: '...roughness.png',
},
```

Карты для бронзового доспеха лежат в `assets/models/armor_tex/`, но в демо
НЕ подключены: чистый metalness=1 под нашим светом пересвечивает модель
(металл отражает белую комнату окружения). Родной материал из FBX вместе
со `scene.environment` (RoomEnvironment, добавлен в обе сцены) выглядит
лучше. Если захотите вернуть PBR-карты — уменьшайте `envMapIntensity`
и яркость света.

⚠️ Модели Meshy очень плотные (этот доспех — 320 тыс. вершин, 28 МБ).
Для телефонов прогоните через Blender: Decimate до ~10-20 тыс. вершин
и экспорт в GLB — будет меньше мегабайта без видимой потери качества.

## Как добавить локацию

Положите картинку в `assets/backgrounds/` и добавьте запись в `LOCATIONS`
(`js/main.js`):

```js
cave: {
  name: 'Пещера',
  image: 'assets/backgrounds/cave.jpg',   // или css: 'linear-gradient(...)'
  actions: [
    { label: 'Выйти к деревне', goto: 'village' },
    { label: 'Охота на разбойника', hunt: true },
    { label: 'Осмотреть алтарь' },        // без поля — заглушка-тост
  ],
},
```

Картинка локации показывается и на экране локации, и как фон арены в бою,
начатом из неё (`arena.setBackground` — CSS-слой под прозрачным канвасом,
подойдёт любой формат и даже `css: 'linear-gradient(...)'`). Картинку лучше
брать «пейзажную», с линией горизонта в нижней трети — персонажи стоят у
нижнего края.

## Замечания по моделям

- **GLB предпочтительнее FBX** для продакшена: меньше и быстрее. Ваш
  `fighter.fbx` (18 МБ) стоит конвертировать в GLB через Blender
  (File → Export → glTF 2.0, формат Binary, +Compression) — выйдет ~3-5 МБ.
  Текущие GLB от mixamo-mini не годятся: в них нет UV-координат и текстур
  (поэтому демо использует FBX).
- В `tools/` лежат python-скрипты: извлечение текстур из FBX
  (`extract_textures.py`) и упаковка их в GLB (`repack_glb.py`).
- **`vendor/three/.../FBXLoader.js` пропатчен** (метки `PATCH(arena-game)`).
  Штатный FBXLoader предполагает, что ключи анимационных кривых стоят
  в одинаковые моменты по всем осям; у оптимизированных экспортов
  (mixamo-mini) это не так, из-за чего анимация дёргалась ступеньками,
  а вращения перемешивались. Патч добавляет честную линейную интерполяцию
  по каждой оси. При обновлении three.js патч нужно перенести.

## Telegram WebApp

`index.html` уже подключает `telegram-web-app.js` и вызывает
`ready()/expand()`. Достаточно отдать игру по HTTPS и прописать URL у бота
(@BotFather → Bot Settings → Menu Button / Web App). Вне Telegram скрипт
ничего не делает.
